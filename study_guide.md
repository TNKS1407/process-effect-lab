# Process Effect Lab 学習ガイド

このガイドは、モデルを「当てる」ためではなく、**パラメータを変えたときに何が起きるかを理解する**ためのものです。数式を暗記するより、まずは動かして観察してください。

## 0. 共通の進め方

### 良い使い方

1. 1 回に変えるパラメータは 1 つだけにする。
2. 結果を 3 つの観点で見る。
   - 予測精度：RMSE、MAE、R²
   - モデルの安定性：係数、係数のばらつき、条件数、VIF
   - データの状態：外れ値、レンジ変更、周期、欠損、下限張り付き、複数モード
3. 「なぜそうなるか」を 1 行でメモする。
4. その現象が現場では何に対応するかを考える。

### 悪い使い方

- RMSE だけを見て「このモデルが最強」と決める。
- クロスバリデーションの最小値だけを機械的に信じる。
- 係数の大きさだけで「重要変数」を決める。
- 生データを見ずにモデル構築へ進む。

## 1. 単回帰：誤差・相関・外れ値を変える

### 触るパラメータ

| 変えるもの | 増やすと何が起きるか | 見る指標 |
|---|---|---|
| `y_noise` | y 側のばらつきが増え、相関・R² が下がる。標準化後の OLS 傾きも相関に近づくので小さくなる。 | corr, slope, R² |
| `x_noise` | x が測定誤差を含む。OLS は x を正確と仮定するため、傾きが真値からずれやすい。 | ols_slope, tls_slope |
| `outlier_strength` | 少数の点が回帰線を引っ張る。外れ値の確認なしに回帰すると危険。 | RMSE, 散布図 |
| `true_slope` | 真の関係の傾きが変わる。ただしノイズが大きいと推定傾きは真値から離れる。 | ols_slope |

### 理解したいこと

OLS は「縦方向の誤差」を小さくする方法です。x と y を平等に扱っているわけではありません。標準化した単回帰では、回帰係数は相関係数になります。つまり、散布図の真ん中を斜めに突っ切る線が常に出るわけではありません。

### 練習

- `y_noise=0.1` と `y_noise=1.5` を比較し、標準化傾きがどう変わるか見る。
- `x_noise` を上げ、OLS 線と PCA/TLS 的な線の違いを見る。
- 外れ値を入れた状態で、RMSE と散布図の印象がどう変わるか見る。

## 2. 多重共線性・PLS：相関の強さと潜在変数数を変える

### 触るパラメータ

| 変えるもの | 増やすと何が起きるか | 見る指標 |
|---|---|---|
| `collinearity` | 入力変数が似た動きをし、係数が不安定になる。条件数・VIF が上がる。 | condition number, VIF, coef_std |
| `pls_components` | 少なすぎると情報不足、多すぎると OLS に近づき多重共線性の悪さが戻る。 | CV_RMSE, 係数, test RMSE |
| `ridge_alpha` | 係数の暴れを抑える。ただし大きすぎると必要な信号まで縮める。 | 係数, RMSE |
| `n_train` | サンプル数が増えると推定は安定しやすいが、強い共線性があると問題は残る。 | coef_std, VIF |
| `y_noise` | 出力ノイズが大きくなり、係数も予測も不安定になる。 | RMSE, R² |

### 理解したいこと

入力変数同士が強く相関していると、見かけ上は予測できていても、係数の解釈が壊れます。係数が大きいから重要、という読み方は危険です。PLS は入力を潜在変数に圧縮し、y と関係の強い方向を使うことで、多重共線性に強くなります。ただし潜在変数を増やしすぎると OLS に戻っていきます。

### 練習

- `collinearity=0.2` と `0.98` を比較する。
- `pls_components=1,2,全変数数` を比較する。
- CV の最小値と、係数の安定性が一致しないケースを探す。

## 3. JIT 型モデル：近さ・k・帯域幅を変える

### 触るパラメータ

| 変えるもの | 増やすと何が起きるか | 見る指標 |
|---|---|---|
| `k` | 使う近傍データが増える。ノイズには強くなるが、局所性が薄れて別状態のデータも混ざる。 | effective_n, error |
| `bandwidth` | 重みの減り方が緩くなる。大きいほど全体モデルに近づく。小さいほど局所的になる。 | effective_n, error |
| `wear_current` | 劣化・メンテナンス状態の違いを表す。似た状態の過去データが少ないと予測が悪くなる。 | nearest_distance |
| `drift_strength` | 時間・劣化による状態変化が強くなり、固定モデルが外れやすくなる。 | global PLS vs LW-PLS |
| `nonlinearity` | 非線形性が強くなり、局所モデルが有利になることがある。 | method ranking |

### 理解したいこと

JIT 型モデルは、事前に 1 つの固定モデルを作るのではなく、現在の状態に近い過去データを使ってその場で予測します。急な状態変化やメンテナンスを含むデータでは、固定モデルより強い場合があります。ただし「近い」の定義が悪いと、近傍選択そのものが失敗します。

### 練習

- `k=3` と `k=100` を比較する。
- `bandwidth` を小さくしすぎて、予測が不安定になるケースを見る。
- `wear_current` を 0.1、0.5、0.9 にして、近い過去データがあるか見る。

## 4. グレーボックス：物理モデルのズレと補正可能性を変える

### 触るパラメータ

| 変えるもの | 増やすと何が起きるか | 見る指標 |
|---|---|---|
| `physics_bias` | 物理モデルの系統誤差が大きくなる。物理モデル単独の RMSE が悪化する。 | Physical only RMSE |
| `parameter_drift` | 物理パラメータが運転状態で変わる。Serial 型が効きやすくなる。 | Serial RMSE |
| `residual_strength` | 物理モデルが説明しきれない残差が大きくなる。Parallel / Combined が効きやすい。 | Parallel, Combined RMSE |
| `residual_observability` | 残差が測定変数から説明しやすくなる。高いほど統計補正が効く。 | correction gain |
| `noise` | 観測ノイズが増える。複雑な補正モデルは過学習しやすくなる。 | test RMSE |

### 理解したいこと

物理モデルは構造がわかる強みがありますが、単純化や未知パラメータのためにズレます。統計モデルはそのズレを補えます。ただし、補正したい残差が観測変数から説明できない場合、統計モデルを足しても改善しません。

### 練習

- `residual_observability=0` と `1` を比較する。
- `parameter_drift` を上げ、Serial 型がいつ有利になるか見る。
- `noise` を上げ、Combined が常に勝つわけではないことを確認する。

## 5. 転移学習：ターゲットデータ数とドメイン差を変える

### 触るパラメータ

| 変えるもの | 増やすと何が起きるか | 見る指標 |
|---|---|---|
| `n_target` | ターゲットデータが増え、Target only が強くなる。転移の必要性は下がる。 | Target only RMSE |
| `domain_gap` | ソースとターゲットの違いが大きくなる。素朴な転移は悪化しやすい。 | Naive transfer RMSE |
| `transfer_weight` | ソースデータの影響が強くなる。近いドメインでは有利、遠いドメインでは害になる。 | transfer RMSE |
| `n_common` | 共通センサーが増えると、ドメイン間の橋渡しがしやすくなる。 | FEHDA-like RMSE |
| `n_target_unique` | ターゲット固有の重要変数が増える。ターゲットデータが少ないと学習が難しい。 | Target / transfer 比較 |

### 理解したいこと

転移学習は「過去の似たデータ」を使う方法です。ターゲットデータが少ないほど価値があります。しかし、似ていないソースを強く混ぜると逆効果になります。共通変数と固有変数を分けて特徴空間を作ると、単純に全部混ぜるより安全になります。

### 練習

- `n_target=5,10,50` を比較する。
- `domain_gap=0.1` と `1.0` を比較する。
- `transfer_weight` を 0, 0.5, 1.0, 2.0 にして、負の転移が出る条件を探す。

## 6. 生データ診断：異常の種類を変える

### 触るパラメータ

| 変えるもの | 増やすと何が起きるか | 見る指標 |
|---|---|---|
| `outlier_strength` | 外れ値が目立つ。回帰や PCA の軸が引っ張られやすい。 | outlier_rate |
| `range_shift` | 前半と後半の水準が変わる。運転条件・測定レンジ・品種変更を疑う。 | range_shift_score |
| `trend_strength` | 経時変化や劣化が強くなる。固定モデルの精度劣化につながる。 | trend_score |
| `cycle_strength` | 周期変動が強くなる。ランダム分割の CV が甘くなることがある。 | periodicity_score |
| `missing_rate` | 欠損が増える。欠損の理由を確認しないと危険。 | missing_rate |
| `lower_clip` | 下限張り付きが増える。センサー下限・検出限界の可能性。 | lower_clip_rate |
| `two_modes` | ヒストグラムの山が分かれる。複数モード・複数品種が混ざっている可能性。 | peak_count |

### 理解したいこと

モデルを作る前に、生データを見てください。外れ値、レンジ変更、周期変動、欠損、下限張り付き、複数モードは、モデル選択以前の問題です。これを放置すると、どんな高度なモデルでも役に立ちません。

### 練習

- `range_shift` を入れて、全期間を 1 つのモデルで扱ってよいか考える。
- `cycle_strength` を上げ、時系列データをランダム分割する危険性を説明する。
- `two_modes` を上げ、ヒストグラムの山が分かれる様子を見る。

## 7. 最後の確認問題

1. 多重共線性が強いとき、テスト RMSE が小さければ係数解釈も信頼できるか。
2. PLS の潜在変数数を増やし続けると、なぜ OLS に近づくのか。
3. JIT 型モデルで `k` を小さくしすぎると何が起きるか。
4. グレーボックスで、統計補正が効かない残差とはどんな残差か。
5. 転移学習で、ソースデータをたくさん使うほど常に良くなるか。
6. 生データに周期変動がある場合、普通のランダム K-fold CV にはどんな落とし穴があるか。
